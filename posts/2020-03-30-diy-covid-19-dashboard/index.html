DIY Covid-19-Dashboard mit R und Shiny

Ein Dashboard-Beispiel mit dynamischen Daten aus Google Sheets.

Eero Kuusisto-Gussmann true
03-29-2020

Einleitung

Mittlerweile gibt es sicherlich schon genügend verschiedene Dashboards mit Covid-19-Daten aus aller Welt. Da es mir aber in der Quarantäne etwas langweilig wurde, wollte ich auch selbst eins erstellen, um u.a. Shiny, googlesheets4 und Plotly besser kennenzulernen.

Passende Daten finden

Die offiziellen Daten von Robert Koch Institute werden leider ziemlich langsam aktualisiert. Deutlich aktueller sind die durch Crowdsourcing gesammelten Fallzahlen von Risklayer, die u.a. auf Twitter täglich veröffentlicht werden. Sie nutzen als Werkzeug eine Google Tabelle, die von aktiven Freiwilligen direkt mit den Daten aus den Landkreisen aktualisiert werden. Als erstes müssen wir also die Daten aus dieser Tabelle mit googlesheets4 herunterladen, die dafür viele Möglichkeiten anbietet. In diesem Fall ist die Tabelle öffentlich zugänglich und wir brauchen nur den ID-Key, der freundlicherweise ein Teil des Links bildet.


library(googlesheets4)
library(tidyverse)

# googlesheets4 ohne Einloggen nutzen 
sheets_deauth()

# ID von der Tabelle speichern
risklayer <- "1wg-s4_Lz2Stil6spQEYFdZaBEp8nWW26gVyfHqvcl8s"
    
# Daten von den einzelnen Bundesländern aus dem Blatt 'Curve2' einlesen
landdata_raw <- sheets_read(risklayer,
                            sheet = "Curve2",
                            range = cell_rows(31:47))

# Bevölkerungdaten aus dem Blatt 'Statistics' als Vektor einlesen und in ein tibble kombinieren
bevoelk <- tibble(Bundesland = pull(sheets_read(risklayer,

                                                range = "B1:B17")),
                  Bevoelkerung = pull(sheets_read(risklayer,

Dann müssen wir noch landdata_raw einmal “rotieren”, damit die Daten in einem “tidy” Format sind. Aktuell sind die täglichen Zahlen als Spalten vorhanden, aber die Bearbeitung ist einfacher, wenn eine Beobachtung immer eine Zeile bildet. Das heißt jedes Bundesland bekommt eine Zeile pro Tag mit der aktuellen Zahl der Infizierte. Darüber hinaus möchte ich die Einwohnerzahlen dabei haben, damit ich z.B. Infizierte pro 100000 Einwohner berechnen kann.


# Fürs Formatieren des Datums
library(lubridate)

landdata <- landdata_raw %>%
        # Daten 'rotieren', Spaltennamen in 'Datum' und Werte in 'Infizierte' speichern
        pivot_longer(-Bundesland, names_to = "Datum", values_to = "Infizierte") %>% 
        # Das Datum richtig formatieren und leere Zeilen entfernen
        mutate(Datum = as_date(Datum, tz = "Europe/Berlin", format = "%e.%m.")) %>% 
                drop_na()

# Mit Bevölkerungsdaten ergänzen: 
landdata <- left_join(landdata, bevoelk, "Bundesland")

Daten visualisieren

Dann generieren wir ein interaktives Liniendiagramm mit Plotly. Auf die X-Achse kommt das Datum, auf die Y-Achse die Zahl der Infizierten, die nach Bundesland separat dargestellt werden sollen.


library(plotly)

plot <- landdata %>% 
        plot_ly(x =  ~ Datum,
                y =  ~ Infizierte,
                color =  ~ Bundesland,
                type = 'scatter',
                mode = 'lines',
                # Das hier ist nur für bessere Formatierung:
                text = ~ Bundesland,
                hovertemplate = paste("<b>%{text}</b><br>",
                                      "%{yaxis.title.text}: %{y:d}<br>",
                                      "%{xaxis.title.text}: %{x}",
                                      "<extra></extra>")
                )

Der Unterschied zu anderen Diagrammen, die man z.B. mit ggplot2 generiert, ist, dass sie interaktiv sind. Die Legende kann man z.B. anklicken, um Länder ein- bzw. ausblenden, was mit einem einfachen Diagramm viele verschiedene Vergleiche und Darstellungen ermöglicht.

Figure 1: Infizierte pro Tag in den Bundesländern

Dann könnten wir beispielhaft noch ein horizontales Balkendiagramm mit den aktuellsten Zahlen im Verhältnis zu den Einwohnern in den Bundesländern generieren.


pro1000plot <- landdata %>% 
                  # nur Zeilen mit dem "größten" Datum auswählen
                  filter(Datum == max(Datum)) %>% 
                  # Eine Spalte mit pro 1000 Einwohner berechnen
                  mutate(pro1000 = Infizierte / Bevoelkerung *1000) %>% 
                  arrange(pro1000) %>% 
                  plot_ly(x = ~pro1000, y = ~Bundesland,
                          type = 'bar', color = ~pro1000,
                          # horizontal, mit Farben aus 'RColorBrewer'
                          colors = "YlOrRd", orientation = 'h',
                          text = ~Bundesland,
                          hovertemplate = paste("<b>%{text}</b><br>",
                                                "%{x:.2f} %{xaxis.title.text}",
                                                "<extra></extra>")) %>% 
                  # Balken nach größe sortieren und die Labels umnennen
                  layout(yaxis = list(categoryorder = "array", categoryarray = ~pro1000,
                                      title = "Bundesland"),
                         xaxis = list(title = paste0("Infizierte pro 1000 Einwohner ", "(", 
                                                     format(max(landdata$Datum), "%d.%m.%Y"), ")"))
                         ) %>%
                  # Legende ausblenden
                  hide_colorbar()

Webapplikation mit Shiny entwickeln

Das Problem mit diesen Diagrammen ist aber, dass sie hier nicht aktualisiert werden können, da diese Webseite statisch ist. Um das zu ermöglichen, braucht man eine Webapplikation mit Shiny, die auf einem Shiny-Server läuft. Ich zeige hier nur eine sehr einfache Variante von einer Applikation, aber auch der Code für das “fertige Produkt” ist auf meinem GitHub verfügbar. Die Bausteine für eine Shiny-Applikation sind server und ui.

UI

Erst müssen wir entscheiden, was wir eigentlich den Benutzern zeigen möchten. Da es ein Dashboard werden sollte, nutze ich hier die Shinydashboard-Erweiterung für die Benutzeroberfläche, die aus drei Elementen besteht: Header, Sidebar und Body.


ui <- dashboardPage(
              dashboardHeader(<hier kommt unser Titel>),
              dashboardSidebar(<hier die Navigation),
              dashboardBody(<und hier der Inhalt>)
)

Eine Navigation brauche ich für dieses Beispiel erst mal nicht, aber die anderen Elemente schon. Ich nehme also die Outputs von den Diagrammen und bringe sie auf zwei dynamischen Spalten an, innerhalb einer dynamischen Zeile. Damit kann die Benutzeroberfläche an die Auflösung bzw. Größe des Endgeräts angepasst werden.


ui <- dashboardPage(
              dashboardHeader(titel = "Covid-19"),
              dashboardSidebar(),
              dashboardBody(fluidRow(
                                  column(width = 8,
                                         box(plotlyOutput("plot"),
                                             width = NULL)
                                        ),
                                  column(width = 4,
                                         box(plotlyOutput("pro1000plot"),
                                             width = NULL)
                                         )
                                    )
                            )
                    )

Server

Auf der “Serverseite” müssen wir nur dafür sorgen, dass die Daten herunterladen und die Diagramme generiert werden. Das heißt wir müssen nur unseren Code von vorher kopieren und die Ergebnisse zu den richtigen Outputs mit der Funktion renderPlotly() steuern.


server <- function(input, output) {
  
          # Zum Beispiel hier könnte man die Daten einlesen
          
          output$plot <- renderPlotly({
            
          landdata %>% 
                plot_ly(x =  ~ Datum,
                        y =  ~ Infizierte,
                        color =  ~ Bundesland,
                        type = 'scatter',
                        mode = 'lines',
                        text = ~ Bundesland,
                        hovertemplate = paste("<b>%{text}</b><br>",
                                              "%{yaxis.title.text}: %{y:d}<br>",
                                              "%{xaxis.title.text}: %{x}",
                                              "<extra></extra>")
                        )  
            
          })
          
          output$pro1000plot <- renderPlotly({
            
          landdata %>% 
                  filter(Datum == max(Datum)) %>% 
                  mutate(pro1000 = Infizierte / Bevoelkerung *1000) %>% 
                  arrange(pro1000) %>% 
                  plot_ly(x = ~pro1000, y = ~Bundesland,
                          type = 'bar', color = ~pro1000,
                          colors = "YlOrRd", orientation = 'h',
                          text = ~Bundesland,
                          hovertemplate = paste("<b>%{text}</b><br>",
                                                "%{x:.2f} %{xaxis.title.text}",
                                                "<extra></extra>")) %>% 
                  layout(yaxis = list(categoryorder = "array",
                                      categoryarray = ~pro1000,
                                      title = "Bundesland"),
                         xaxis = list(title = paste0("Infizierte pro 1000 Einwohner ",
                                                     "(", 
                                                     format(max(landdata$Datum),
                                                            "%d.%m.%Y"), 
                                                     ")"))
                         ) %>%
                  hide_colorbar()  
            
          })
  
  
}

Fertiges Dasboard

Letztlich bleibt nur noch übrig die App mit shinyApp(ui, server) zu starten. Nach ein paar Schönheitsreparaturen und einigen weiteren Funktionen könnte die App aussehen wie in Abbildung 3. Den noch ziemlich primitiven1 Quelltext dieser abgebildeten App findet ihr auf meinem GitHub.

Figure 3: Fertiges Dashboard (Stand 19.04.2020)

Mit den gleichen Prinzipien könnte man z.B. eine sich automatisch aktualisierende Auswertung einer Umfrage erstellen, die mit Google Forms durchgeführt werden. Auch schreiben auf Google Tabellen ist möglich. Wenn also die Formeln nicht genug Funktionen anbieten oder zu komplex werden, kann man die Daten mit R bearbeiten und dann zurück auf Google Tabellen speichern. Das sind beide Funktionen, die ich in einem kommenden Projekt benötigen werde, was auch der Hauptgrund für diese kleine Übung war. Vielen Dank fürs Lesen!
